## 1. Product & Configuration

### `loan_products`

Loan product catalogue. Each product defines the lending envelope (amount range, tenor, type, channel). All downstream applications reference a product.

| Column | Type | Nullable | Description |
|---|---|---|---|
| `id` | UUID | NOT NULL | Primary key. Generated by `gen_random_uuid()`. |
| `code` | TEXT | NOT NULL | Unique short code. e.g. `PIL_CONSUMER`, `PIL_SAVINGS`. Used in APIs and reports. |
| `name` | TEXT | NOT NULL | Display name in Vietnamese. e.g. `Vay t√≠n ch·∫•p ti√™u d√πng`. |
| `loan_type` | `loan_type` | NOT NULL | `UNSECURED` (t√≠n ch·∫•p) or `SECURED_SAVINGS` (c·∫ßm c·ªë ti·∫øt ki·ªám). |
| `min_amount` | NUMERIC(20,2) | NOT NULL | Minimum loan amount in VND. Must be > 0. |
| `max_amount` | NUMERIC(20,2) | NOT NULL | Maximum loan amount in VND. Must be ‚â• `min_amount`. |
| `min_term_months` | INTEGER | NOT NULL | Minimum loan tenor in months. |
| `max_term_months` | INTEGER | NOT NULL | Maximum loan tenor in months. Must be ‚â• `min_term_months`. |
| `channel` | `loan_channel` | NOT NULL | Channel where this product is available. Default `MOBILE_APP`. |
| `is_active` | BOOLEAN | NOT NULL | If FALSE, the product is no longer available for new applications. |
| `created_at` | TIMESTAMPTZ | NOT NULL | Row creation timestamp. |
| `updated_at` | TIMESTAMPTZ | NOT NULL | Auto-updated by trigger on every UPDATE. |

### `eligibility_rules`

Per-product eligibility rules evaluated by the decisioning engine. Hard-block rules auto-decline on failure; soft rules generate a manual review flag.

| Column | Type | Nullable | Description |
|---|---|---|---|
| `id` | UUID | NOT NULL | Primary key. |
| `product_id` | UUID FK | NOT NULL | References `loan_products.id`. |
| `rule_code` | TEXT | NOT NULL | Unique rule identifier within the product, e.g. `AGE_MIN`, `CIC_GROUP_MAX`. |

| `operator` | TEXT | NOT NULL | Comparison operator: `>=`, `<=`, `=`, `<`, `>`, `IN`, `NOT_IN`. |
| `threshold_value` | TEXT | NOT NULL | The threshold as a string, e.g. `18`, `2`, `0.50`. |
| `is_hard_block` | BOOLEAN | NOT NULL | TRUE = auto-decline if rule fails; FALSE = soft flag for manual review. |
| `decline_reason_code` | TEXT | NULL | Machine-readable decline reason emitted in `decisioning_runs`. |

## Business Rules

| ID | Rule | Enforced by |
|---|---|---|
| **BR-01** | A `loan_application` may only link to a `kyc_session` with `status = 'PASSED'`. | Application layer / CHECK |
| **BR-02** | A `loan_application` may only link to a `bank_account` with `is_verified = TRUE`. | Application layer |
| **BR-03** | `CIC_INQUIRY` customer consent must be recorded with `granted = TRUE` before a `cic_inquiries` row is created. | Application layer |
| **BR-04** | `cic_inquiries.cic_group` IN (3, 4, 5) causes `decisioning_runs.decision = 'DECLINED'` via the `CIC_GROUP_MAX` eligibility rule. | `eligibility_rules` + engine |
| **BR-05** | `sanctions_checks.result = 'HIT'` causes auto-decline and triggers SAR filing. Customer `status` set to `BLACKLISTED`. | Application layer |
| **BR-06** | `fraud_checks.result = 'BLOCK'` causes auto-decline and customer `status` set to `BLACKLISTED`. | Application layer |
| **BR-07** | `credit_scores.dti_ratio` must be ‚â§ 0.50 to qualify for `decision = 'APPROVED'`. | `eligibility_rules` (DTI_MAX) |
| **BR-08** | A `loan_offer` may not be accepted after `valid_until`. | CHECK constraint + application layer |
| **BR-09** | A `loan_contract` PDF and `signed_at` may not be null when `status = 'ACTIVE'`. | Application layer |
| **BR-10** | High-value loans (approved_amount ‚â• 300,000,000 VND) require `esign_events.method = 'QUALIFIED_ESIGN'`. | Application layer / Lu·∫≠t GDƒêT 2023 |
| **BR-11** | `repayment_schedules` rows are generated immediately when `disbursements.status` transitions to `COMPLETED`. | Application layer (event-driven) |
| **BR-12** | `overdue_records` rows are created by a daily batch job for any `repayment_schedules` row where `due_date < CURRENT_DATE AND status = 'SCHEDULED'`. | Scheduled job |
| **BR-13** | `debt_classifications` rows are created monthly and reported to CIC. `reported_at` must be set within 5 business days of month-end per TT02/2013. | Scheduled job / compliance |
| **BR-14** | `audit_log` rows are immutable ‚Äî `UPDATE` and `DELETE` are blocked by database triggers. | `trg_audit_log_no_update`, `trg_audit_log_no_delete` |
| **BR-15** | `collaterals.maturity_date` must be ‚â• `CURRENT_DATE` at application time (CHECK constraint relaxed for `is_released = TRUE`). | CHECK constraint |
