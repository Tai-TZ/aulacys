## What changed & why

audit / los / application tests no longer silently skip green without a DB in CI. Local `docker-compose.db.yml` + `scripts/test-db.*` + `REQUIRE_DB=1` (skip → fail). CI job `db-services` matrix runs all three against a Postgres service container.

## Files touched

- `docker-compose.db.yml` — Postgres-only for tests
- `scripts/test-db.sh`, `scripts/test-db.ps1`, root `Makefile` (`test-db` / `db-up` / `db-down`)
- `services/{audit,los,application}-svc/tests/*` — `connect_timeout=3` + `_unavailable` / `REQUIRE_DB`
- `.github/workflows/ci.yml` — job `db-services`
- `docs/CONFIG.md` + service READMEs

## How to run / verify

```powershell
docker compose -f docker-compose.db.yml up -d --wait
.\scripts\test-db.ps1 -NoUp
# expect: test-db OK (audit + los + application) — no skips
```
